# Dokumen Perencanaan Proyek (Revisi)
## Tugas Besar KAIT II — Website E-Commerce Fungsional dengan Integrasi Strategi Bisnis Modern

**Stack:** Pure HTML5 + CSS3 (Flexbox/Grid) + JavaScript (ES6+) Vanilla
**Penyimpanan data:** `localStorage` (cart & preferensi), data produk di-hardcode dalam file JS (`products.js`)

# 1. BRD — Business Requirements Document

## 1.1 Latar Belakang
Tugas Besar KAIT II (bobot 40% nilai akhir) mengharuskan mahasiswa membangun prototype website e-commerce yang menggabungkan implementasi teknis (HTML/CSS/JS) dengan pemahaman strategi bisnis nyata. Niche bisnis bebas dipilih dengan persetujuan dosen. Dokumen ini memakai **"Toko UMKM"** sebagai contoh generik (silakan ganti dengan niche pilihanmu — kopi specialty, fashion lokal, kerajinan tangan, dll.).

## 1.4 Ruang Lingkup

**In-Scope:**
- Katalog produk statis (minimal 8-10 produk, data di file JS)
- Filter & search produk (kategori, harga, nama) — diproses di JS, real-time tanpa reload
- Keranjang belanja tersimpan di `localStorage` (persist walau browser di-refresh)
- Checkout dengan form + simulasi pembayaran (tanpa transaksi nyata)
- Dokumentasi bisnis (Business Overview) & teknis (README)
- Desain responsif dengan gaya glassmorphism

**Out-of-Scope:**
- Login/akun pengguna sungguhan
- Database & backend server
- Payment gateway asli
- Panel admin (karena tidak ada backend untuk autentikasi/CRUD aman)

## 1.6 Asumsi
- Data produk cukup realistis meski hardcode (bukan asal-asalan), agar Business Overview terasa kredibel.
- Cart yang tersimpan di `localStorage` dianggap cukup mewakili "keranjang belanja fungsional" sesuai brief.
- Pembayaran murni simulasi visual (redirect ke halaman "Pembayaran Berhasil" dummy).

## 1.7 Risiko
| Risiko | Dampak | Mitigasi |
|---|---|---|
| Data hilang jika user clear browser cache | Cart kosong lagi | Wajar untuk prototype, jelaskan sebagai batasan di README |
| Filter/search lambat jika logic tidak efisien | UX terganggu | Gunakan array method JS (`filter`, `map`) yang efisien, hindari manipulasi DOM berlebihan |
| Kurang variasi produk membuat katalog terlihat kosong | Nilai UX turun | Pastikan minimal 8-10 produk dengan gambar & deskripsi menarik |

# 2. PRD — Product Requirements Document

## 2.1 Ringkasan Produk
**Nama produk (working title):** *TokoKita* (sesuaikan dengan niche pilihanmu)
**Deskripsi:** Website e-commerce statis single-page atau multi-page sederhana, dibangun murni dengan HTML5, CSS3, dan JavaScript ES6+. Cart disimpan di `localStorage` browser sehingga tetap ada walau halaman di-refresh. Gaya visual glassmorphism dengan elemen transparan berlapis blur.

## 2.2 Tech Stack
| Layer | Teknologi |
|---|---|
| Markup | HTML5 semantik |
| Styling | CSS3 (Flexbox & Grid) — Tailwind CSS diperbolehkan sebagai utility class, dijelaskan alasan di README (efisiensi styling glassmorphism) |
| Logic/Interaktivitas | JavaScript ES6+ vanilla (tanpa framework: no React/Vue/jQuery) |
| Data Produk | File `products.js` (array of objects, hardcode) |
| Penyimpanan Cart | `localStorage` (key: `cart`, format JSON) |
| Analytics | Script Google Analytics dummy (gtag.js placeholder) |
| Version Control | Git + GitHub |
| Deployment | GitHub Pages |

## 2.3 Desain — Glassmorphism Guidelines
- Background: gradient lembut atau image dengan overlay blur
- Card/Panel: `background: rgba(255,255,255,0.2); backdrop-filter: blur(12px); border: 1px solid rgba(255,255,255,0.3); border-radius: 16px;` (atau utility Tailwind setara: `bg-white/20 backdrop-blur-md border border-white/30 rounded-2xl`)
- Warna brand: 1 primary + 1 accent, hindari warna solid berlebihan agar efek glass menonjol
- Font modern (Inter/Poppins via Google Fonts CDN)
- Konsisten radius & spacing di semua card/komponen
- Subtle animation: transisi hover pada button/card (`transition: all 0.3s ease`)

## 2.4 User Persona
1. **Customer (Bu Sari, 34 th)** — belanja produk UMKM cepat, ingin filter harga jelas, cart tidak hilang saat pindah halaman.

## 2.5 Struktur Halaman & Fitur

| ID | Fitur | Deskripsi | Prioritas |
|---|---|---|---|
| F1 | Navbar + Hero Banner | Navigasi ke Katalog/Cart, CTA utama | Must |
| F2 | Halaman Katalog Produk | Render 8-10 produk dari `products.js` via JS (`.map()`/`forEach`) ke DOM | Must |
| F3 | Filter & Search | By kategori, rentang harga, nama — real-time filter di client (JS `.filter()`) | Must |

| F9 | Smooth Scroll & Animasi | Scroll halus antar section, hover animation pada card | Should |

## 2.6 Detail Teknis Fitur Kunci

**Struktur data produk (`products.js`):**
```js
const products = [
  {
    id: 1,
    name: "Kopi Arabika Gayo 250g",
    category: "Kopi",
    price: 45000,
    stock: 20,
    image: "assets/images/kopi-gayo.jpg",
    description: "Kopi arabika single origin dari Gayo, Aceh."
  },
  // ...minimal 8-10 produk
];
```

**Struktur data cart di `localStorage`:**
```js
// key: "cart"
[
  { productId: 1, name: "Kopi Arabika Gayo 250g", price: 45000, qty: 2 },
  { productId: 3, name: "Teh Melati Premium", price: 25000, qty: 1 }
]
```

**Fungsi JS inti yang perlu dibuat:**
- `renderProducts(list)` — render grid produk ke DOM
- `filterProducts(keyword, category, priceRange)` — return array hasil filter
- `addToCart(productId, qty)` — update localStorage
- `getCart()` / `saveCart(cart)` — helper baca/tulis localStorage
- `calculateTotal(cart)` — hitung subtotal & total
- `validateCheckoutForm(formData)` — validasi input sebelum submit
- `renderCartBadge()` — update jumlah item di ikon cart navbar

## 2.7 Non-Functional Requirements
- **Responsive**: mobile (≤640px), tablet (641-1024px), desktop (>1024px) via media query/Tailwind
- **Performance**: gambar dioptimasi (compressed, lazy-load pakai atribut `loading="lazy"`)
- **Browser compatibility**: modern browser (Chrome, Firefox, Edge terbaru)
- **Kode bersih**: JS dipisah per file/modul (`products.js`, `cart.js`, `checkout.js`, `main.js`), komentar pada fungsi kompleks
